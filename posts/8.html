I made a discord AI bot for FREE

Hi, i'm Stuf_y and I made a discord AI bot that YOU can use for free.

Recently, I have been wanting to make my own AI. I have been wanting to make an AI smarter, better and faster than most others. I wanted it more reliable and more easy to access and most importantly FREE. So thats when I made....

Tye AI

Tye AI is a discord bot that you can chat to completely FOR FREE. And thats not even the best part. Tye AI is (mostly) open source for ANYONE to host. In this series of blog post, I will go over every little detail of how YOU can host and use it. But first, heres how you can access it.

Accessing Tye

To access Tye, you must FIRST join the discord server. This is crucial so you can message the AI. To join the discord server, click here. Once you have joined the discord server, go to the #subscribe channel and press subscribe. After this, you will get a DM from the bot. You can now talk to the bot.

NOTE: The bot is currently in BETA so may not always work

Hosting Tye

NOTE: To make Tye free for everyone, we had to think carefully about ways around roadblocks that would make us have to pay so it can be quite complicated sometimes!!

To host Tye, you need the following:

  • A platform to host the discord bot on (Recommended: Railway)
  • A server with docker and n8n (Recommended: Oracle Cloud free server)
  • A discord account
  • Basic n8n knowledge
  • A github account

PART 1: Creating the n8n workflow

NOTE: You need n8n setup first

  1. Open your n8n dashboard
  2. Press create workflow
3. Add the 'webhook' node - Add authentication and set your own credentials - Set respond to 'Using respond to webhook node' 4. Connect an 'AI Agent' node to it - Set prompt (user message) to `{{ $json.query.prompt }}` - And create your own system message. Currently there is not a public system prompt for Tye. - Turn on automatically passthrough binary images - Enable fallback model 5. Connect the models you want - If you want free AI models, use `google/gemma-4-26b-a4b-it:free` for backup and `openai/gpt-oss-120b:free` for the main model - For setting up openrouter, just create an account then make an api key [here](https://openrouter.ai) 6. Connect the tools you want - You can browse the tool library [here](https://github.com/TyeArtifical/Brains/tree/main/tools) 7. Connect the memory that you want - We recommend simple memory - Set the session id to define below and the sessionid key to `{{ $json.query.sessionid }}` - Set the context window length to 10 8. Connect the 'Respond to webhook' node to the output of the AI agent. - Set 'Respond with' to 'Text' - Set 'Response Body' to `{{ $json.output }}` > Well done: You have now finished setting up n8n

NOTE: To add things to n8n from the GitHub library, copy the JSON then paste it into n8n

PART 2: Setting up the discord bot

  1. Go to the discord developer portal and create an application
    • Get your bot token
    • Customise your bot
  2. Create a discord server and add the bot to the server
    1. Create a discord server
    2. Add a role to the discord server for people who can use Tye. Save the ID for this for later. DO NOT GIVE YOURSELF THIS ROLE YET.
    3. Go to OAuth2 on the discord bot panel
    4. Select "Bot"
    5. Select Administrator as the bot permissions

PART 3: Setting up your bot hosting

NOTE: For this guide, we will be using Railway which is free and simple.

  1. Fork the Backend Repo
  2. Go to Railway and press new project
  3. Press 'Deploy from GitHub'
  4. Select the repo you just forked.
  5. Configure the following environment variables:
  • BOT_TOKEN - Your discord bot token
  • WEBHOOK_BASE_URL - Your production webhook URL from n8n
  • ALLOWED_GUILD_ID - Your discord server ID
  • ALLOWED_ROLE_ID - The role ID that is allowed to use Tye
  • WEBHOOK_USER the username for basic auth on the webhook on n8n
  • WEBHOOK_PASS the password for basic auth on the webhook on n8n
  1. Deploy the code

PART 4: Testing

  1. Give yourself the role that lets you use Tye.
  2. You should get a DM from Tye. You should now be able to chat to it. If you do not get a DM, check the logs on Railway.
  3. Send a message to Tye.
    • If you get an error instantly, something like the credentials are wrong or you have not enabled the workflow for production on n8n.
    • If you get an error after quite a while, it has taken too long. Check the executions on n8n to fix this.

NOTE: If you need help, please feel free to comment below.

Now, you have completed the setup guide for hosting Tye AI and you have your own!
